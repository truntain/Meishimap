CHANGELOG - Plan A: MapLibre + OpenFreeMap cho màn Search

File này ghi lại toàn bộ thay đổi khi triển khai bản đồ cho màn /search. Mục tiêu là để nhóm dễ đối chiếu: file nào được thêm/sửa, tên biến/API nào mới, dữ liệu đi qua những bước nào, và phần nào chưa đụng tới.

Phạm vi: chỉ làm màn Search. Không làm Google Places, không routing/chỉ đường, không import dữ liệu thật từ bên ngoài, không sửa các màn booking/review/detail nếu không cần.

1. Trạng thái trước khi sửa

  • Frontend /search đang dùng dữ liệu mock trong SearchClient.tsx.
  • Khu vực bản đồ bên phải là placeholder tự dựng, chưa có bản đồ thật.
  • Database đã có bảng restaurants với các cột tọa độ latitude, longitude.
  • Backend chưa có module/API cho restaurants.

2. Kế hoạch thay đổi

Hạng mục Dự kiến thay đổi Lý do
Backend Thêm entity/service/controller/module cho restaurants và API GET /restaurants. Lấy dữ liệu thật từ PostgreSQL thay cho mock frontend.
Frontend dependency Thêm maplibre-gl. Render bản đồ vector bằng MapLibre GL JS.
Frontend Search Đổi SearchClient.tsx sang fetch API backend, render MapLibre/OpenFreeMap, marker, popup, flyTo. Biến khu vực map placeholder thành bản đồ thật có tương tác.

5. API và biến mới

Tên Loại Ý nghĩa
GET /restaurants Backend API Trả danh sách nhà hàng từ bảng restaurants.
q Query param Từ khóa tìm kiếm theo tên, tên Nhật, địa chỉ, quận, thành phố, category.
category Query param Lọc nhà hàng theo danh mục, ví dụ sushi, ramen.
sort Query param Sắp xếp theo rating hoặc name.
RestaurantResponse TypeScript type Response camelCase cho frontend: nameJp, imageUrl, hasJapaneseSupport.
maplibre-gl Frontend dependency Thư viện WebGL để hiển thị bản đồ MapLibre/OpenFreeMap.
API_BASE_URL Frontend constant Lấy từ NEXT_PUBLIC_API_BASE_URL, mặc định http://localhost:3001.
MAP_STYLE_URL Frontend constant Style OpenFreeMap: https://tiles.openfreemap.org/styles/liberty.
Restaurant Frontend type Kiểu dữ liệu nhà hàng dùng trong màn Search.
featureCollection Frontend computed data Chuyển danh sách nhà hàng thành GeoJSON để MapLibre render marker/cluster.
focusRestaurant() Frontend function Khi click card/marker, gọi map.flyTo(), set active restaurant và mở popup.
FALLBACK_RESTAURANTS Frontend fallback data Dữ liệu demo dùng khi backend API tạm thời không tải được, giúp màn Search vẫn demo được.

7. Version 2 - Làm bản đồ sát Google Maps hơn

Mục tiêu V2: nâng cảm giác bản đồ chuyên nghiệp hơn mà vẫn giữ Plan A: MapLibre + OpenFreeMap, không cần API key. V2 tập trung vào UI/UX bản đồ: marker dạng pin, hover/click sync, popup card giàu thông tin, dữ liệu demo dày hơn.

7.1. Kế hoạch V2

Hạng mục Dự kiến thay đổi Lý do
Dữ liệu demo fallback Tăng số nhà hàng demo quanh TP.HCM. Map có nhiều marker/cluster hơn, nhìn giống trải nghiệm Google Maps hơn khi API backend chưa có đủ dữ liệu.
Marker Chuyển marker điểm đơn từ circle layer sang symbol marker dạng pin SVG. Marker dạng pin trực quan và gần Google Maps hơn.
Popup Nâng popup thành mini card có tên, rating, địa chỉ, tag và nút đặt bàn. Người dùng xem thông tin nhanh ngay trên map.
Tương tác Hover card highlight marker, click marker scroll tới card, click card flyTo marker. Tăng cảm giác đồng bộ list-map như Google Maps.
CSS Thêm style pin, marker active, popup, hover state và floating panel. Làm giao diện sắc nét hơn, giảm cảm giác placeholder.

8.5. Hotfix V3.1 - Sửa expression MapLibre halo

Vấn đề Cách sửa Kiểm thử
MapLibre báo lỗi runtime ở restaurant-halo vì ['zoom'] bị đặt lồng trong ['case'] cho circle-radius và circle-opacity. Đổi expression thành dạng ['interpolate', ['linear'], ['zoom'], ...] ở tầng đầu, còn ['case'] chọn marker active nằm trong từng mốc zoom. cd frontend && npm.cmd run build: Pass.
